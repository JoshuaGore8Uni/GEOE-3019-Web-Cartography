Emotional Maps of Australia: Out of the Box Web Mapping

The aim of this exercise was to create a map showing the emotions I feel in places across Adelaide using two out-of-the-box mapping solutions, thus comparing their features and ease of use. Popupation density for Australian states is shown in the background, calculated from population and area fields.

ArcGIS Online and Carto are two very different out of the box web mapping solutions. Carto is visually appealing and designed with ease of use in mind. It has a simple, clean feel and often loads much faster than ArcGIS online. Finding where uploaded data and created maps are stored is logical and simple, through the effectively named and easily accessible “dashboard”. ArcGIS online on the other hand has maps and data under the “content” link, which is stuck in the middle of several other links at the top. These include advertising, the ‘map’ link used to create a new map (not obvious until followed) and the ‘scene’ link used to create a 3d scene, also not obvious until followed! Once the main control area for each site has been located, uploading data is fairly straightforward in both platforms. Displaying a calculated value is simple in carto, done through the clearly labelled ‘SQL’ link (though it wasn’t immediately obvious that this could also be used to modify results on retrieval). ArcGIS does not provide any data editing or retrieval tools. Values must be programmatically calculated for both the custom attribute style and the popup, or calculated before uploading data. Creating symbology was straightforward for both platforms. ArcGIS provided more choice in assigning symbology graphically, but carto styles can be modified through the easy to use carto css. Popups where also easier to configure and more graphically appealing in carto. Overall carto is a visually appealing, simple platform for creating web maps. ArcGIS is harder to use, does not create maps with the same visual appeal as carto by default, but as part of a complete GIS system is more powerful for applications focused data rather than visual appeal.
